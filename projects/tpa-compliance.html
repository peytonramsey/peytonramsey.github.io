Problem & Approach

Manual Oversight at Scale

The binding constraint going in: compliance analysts aren't engineers. That ruled out every solution that required a data pull, a query, or a code change to get an answer. The delivery layer had to be entirely self-serve, or it wouldn't get used.

The problem itself was 120K+ insurance policies reviewed manually across 7 AWS data sources. Analysts were reconciling entity mismatches by hand, scoring quality inconsistently across 8 activity types, with no historical data to track whether a TPA was actually improving.

I built the system solo: ETL, scoring engine, dashboard. One early decision shaped everything else: activity-specific weights instead of a single global score. Disbursement reviews and routine transaction checks carry different risk profiles. Weights and pass thresholds are externalized as config, so the team can tune the system as their understanding matures without a code deploy.

WHAT I LEARNED

The technical build was maybe 40% of the job.

The scoring engine worked on my laptop weeks before a compliance analyst could actually use it. Not because of bugs, but because the gap between working code and deployable code is mostly about the people using it. Designing a UI that required no training, handling edge cases my clean test files never exposed, writing queries fast enough not to time out on 100M rows. That was the other 60%.
